# Organization-Wide Development Rules

## Code Quality

### Testing
- All code must have tests
- Minimum 80% coverage
- Tests pass before merging

### Style
- Follow ESLint rules
- Use Prettier formatting
- Consistent naming

## Git Practices

### Commits
- Conventional commit messages
- Atomic commits
- Reference issues

### Branches
- Feature branches
- Keep main stable
- Delete after merge

## Documentation

- README required
- API docs for public functions
- Keep changelogs updated
